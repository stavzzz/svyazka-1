## 1. Цель и результат

**Боль:** планирование съедает внимание каждый день. Перенос одной встречи тянет за собой три. У кого был живой помощник, тот платил ему в основном за ведение календаря.

**Результат:** агент в Telegram ведёт Google Календарь голосом и текстом. Пользователь не открывает календарь руками.

**Критерий готовности:** пройден чек-лист приёмки из раздела 14, все 15 пунктов.

## 3.1 Разделение ответственности — главное правило связки

**Голос и текст нужны только для того, чтобы вызвать команду. Всё остальное делает код.**

### Что делает модель

Ровно одно: превращает фразу пользователя в строгий JSON с интентом и полями. Точка.

```
«поставь встречу с Иваном завтра в 15:00 по Москве»
        ↓
{ "intent": "create", "title": "Иван", "date": "2026-07-23",
  "time_start": "15:00", "source_city": "Москва", "duration_min": "60" }
```

### Что модели ЗАПРЕЩЕНО

| Запрещено | Почему |
|---|---|
| Считать время и разницу часовых поясов | Модель ошибается в поясах, проверено на проде |
| Определять конфликты и пересечения | Нужна точная арифметика интервалов |
| Формулировать ответы пользователю | Ответы обязаны быть одинаковыми всегда |
| Придумывать недостающие поля | Не названо время — переспросить, а не угадать |
| Решать, что делать при накладке | Решение принимает пользователь кнопкой |
| Форматировать даты, время и HTML | Форматирование детерминировано |

### Что делает программный код

- Нормализует интент, распознаёт город по словарю, конвертирует зоны
- Считает диапазоны дат для семи вариантов расписания
- Определяет конфликт по формуле пересечения интервалов
- **Рендерит все до единого сообщения по шаблонам раздела 6**
- Собирает inline-кнопки и разбирает `callback_data`
- Хранит состояние, следит за TTL, защищает от дублей
- Запускает планировщики и проверяет гейты
- Вызывает Workspace MCP и Zoom API

### Что делает Workspace MCP

- Создаёт, читает, обновляет и удаляет события
- Добавляет ссылку Google Meet флагом `add_google_meet`
- Читает и меняет `timeZone` календаря
- Рассылает приглашения участникам

### Проверка правильности сборки

Одна и та же фраза, отправленная десять раз, должна давать **побайтово одинаковый ответ**. Если формулировки плавают, значит ответ формирует модель, и это ошибка сборки.

Семь диапазонов: сегодня · завтра · послезавтра · день недели · эта неделя · следующая неделя · конкретная дата.

- Показать текущую зону

- Напоминание за сутки, за час и за пять минут

- Если время не названо, **переспросить**, не подставлять своё

## 6. Тексты сообщений

> **Все сообщения ниже рендерит программный код по шаблонам.** Модель их не пишет и не переформулирует. Любой текст, попадающий пользователю, собирается подстановкой значений в эти шаблоны.

### Три правила оформления

1. Все сообщения отправляются с `parse_mode: HTML`. Теги: `<b>`, `<i>`, `<a href="">`.
2. Любой текст от пользователя или из Google экранируется:
   ```js
   const esc = (s) => String(s || '').replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;');
   ```
3. Разделитель шапки — ровно 20 символов `━`.

Обозначения: `{{ переменная }}` подставляется в рантайме, `[строка]` выводится только при выполнении условия.

```html
[👥 vasya@mail.com, petya@mail.com]   ← только если есть участники
```

### 6.4 Конфликт времени — единственное сообщение с кнопками

```html
<b>⚠️ Конфликт времени</b>
━━━━━━━━━━━━━━━━━━━━

📌 <b>{{ title }}</b>
🗓 {{ 13 июля 2026 }}
🕙 <b>10:00 – 11:00 МСК</b>

⛔ Пересекается с:
• <b>Планёрка</b> 10:30–11:30 МСК
• <b>Ещё встреча</b> 10:00–10:45 МСК

Что делаем?
```

Кнопки одним рядом:

| Текст | `callback_data` |
|---|---|
| `✅ Всё равно` | `cal:add:{{ pending_key }}` |
| `🔁 Перенести` | `cal:reschedule:{{ pending_key }}` |
| `❌ Отмена` | `cal:cancel:{{ pending_key }}` |

**Протокол:** `cal:<action>:<pending_key>`, где `pending_key` = `<chat_id>_<8 случайных символов>`.
Лимит Telegram 64 байта, кириллицу в payload не класть.
При разборе проверить префикс `cal`, найти отложенное событие, сверить ключ. Не совпало — считать протухшим.

## 7. Поля, которые извлекаются при создании

```json
{
  "intent": "create",
  "title": "Тест-А",
  "date": "2026-05-28",
  "time_start": "11:00",
  "time_end": "12:00",
  "duration_min": "60",
  "attendees": ["a@b.com"],
  "location": "",
  "description": ""
}
```

- `title` — только название, без слов «встреча» и «звонок», кавычки убрать
- `time_start` — в зоне, **как сказал пользователь**. Не названо — пустая строка, **не угадывать**
- Пустой `time_start` — обязательно спросить, не подставлять 00:00
- Длительность по умолчанию 60 минут

## 8. Состояние

| Ключ | Что хранит | Очистка |
|---|---|---|
| `pending_events[chat_id]` | отложенное событие, ждёт кнопки или времени | 1 час |

## 9. Планировщики

| Что | Период | Логика |
|---|---|---|
| Напоминания | каждую минуту | события, стартующие через 4:30–5:30 мин, и аналогично для часа и суток |
| Обновление кэша | каждый час | тянет зону и события из Google |
| План дня | 08:00 | один раз в день |
| План недели | понедельник 09:00 | один раз в неделю |

**Окно напоминания — интервал, а не точка.** Точное «ровно 5 минут» промахивается из-за дрейфа планировщика.

Команды OpenClaw сверять по скиллу `openclaw-docs`. Базовый синтаксис:

```bash
openclaw cron create "0 8 * * *" "Собери мой план на сегодня и пришли списком." --name "План дня"
openclaw cron create "0 9 * * 1" "Собери мой план на неделю и пришли по дням." --name "План недели"
openclaw cron list
openclaw cron run <job-id>
```

```js
// пересечение интервалов
if (evStart < newEnd && evEnd > newStart) → конфликт
```

События со `status === 'cancelled'` пропускаются. Сверка идёт по кэшу, не по живому запросу.

## 11. Настройка Google

### Шаги

1. Открыть https://console.cloud.google.com и выбрать проект
2. Включить Calendar API: https://console.cloud.google.com/apis/library/calendar-json.googleapis.com
3. Экран согласия: https://console.cloud.google.com/apis/credentials/consent — тип «Внешний»
4. Области доступа: `.../auth/calendar` и `.../auth/calendar.events`
5. **Опубликовать приложение** — статус «В эксплуатации»
6. Создать OAuth Client ID типа «Веб-приложение», добавить redirect URI
7. Сохранить Client ID и Client Secret

### Критично

- В статусе **Testing** refresh-токен живёт **7 дней**, лимит 100 тестовых пользователей. Это самая частая причина, по которой агент замолкает через неделю. Лечится переводом в Production.
- Верификация Google не требуется, если приложение для личного использования. Пользователь пройдёт через экран «приложение не проверено».

### Деньги

Calendar API бесплатен, платёжный аккаунт не нужен, платный Workspace не нужен.
Квоты: 10 000 запросов в минуту на проект, 600 в минуту на пользователя, порог 1 000 000 в сутки.
Google предупредил, что плата за превышение суточного порога может появиться позже в 2026 году.

## 12. Настройка Zoom, опционально

1. https://marketplace.zoom.us → Develop → Build App
2. Тип: **Server-to-Server OAuth**, работает без окна входа
3. Scopes: права на встречи, чтение и запись
4. App Credentials: скопировать Account ID, Client ID, Client Secret
5. Activation: нажать Activate

### Деньги и лимиты

Приложение и API бесплатны, `join_url` возвращается и на бесплатном тарифе.
Ограничения бесплатного плана: групповая встреча **40 минут**, до 100 участников, одновременно только одна встреча.
Rate limits для Free: Light 4/сек и 6000/день, Medium 2/сек и 2000/день, Heavy 1/сек и 1000/день.

**Не подтверждено официально:** может ли владелец бесплатного Basic создать Server-to-Server приложение. Запрета в документации нет, но и прямого разрешения тоже. Проверить на живом аккаунте при сборке.

### Google Meet как альтернатива

Ставится флагом `add_google_meet` в Workspace MCP, отдельная настройка не нужна.
Лимиты бесплатного личного аккаунта: один на один до 24 часов, трое и больше — 1 час.
**Не подтверждено официально:** создаётся ли Meet-ссылка через API на бесплатном личном Gmail. Документация ограничений по типу аккаунта не вводит, но предписывает проверять `conferenceProperties.allowedConferenceSolutionTypes` конкретного календаря.

## 14. Чек-лист приёмки

1. Создание с временем → карточка 6.1, событие в Google, участники приглашены
2. Создание без времени → forceReply 6.5, ответ временем → создаётся
3. Создание на занятое время → 6.4 с тремя кнопками
4. Кнопка «Всё равно» → создаётся · «Отмена» → 6.7 · «Перенести» → 6.6
5. Нажать кнопку через час → внятный ответ 6.9, не тишина
6. «Что сегодня / завтра / на этой неделе / 20 июля» → 6.2 и 6.3
7. Встреча в чужой зоне → в карточке **две** строки времени
8. «Переключи зону на Тбилиси» → 6.10, следующее расписание уже в новой зоне
9. «Какая у меня зона» → 6.12
10. «Удали Тест-1 и Тест-2» → обе удалены
11. Удалить несуществующую → 6.15
12. Название совпадает у двух встреч → 6.16
13. За 5 минут до встречи → 6.18, ровно один раз
14. В 08:00 → 6.19, ровно один раз за день
15. Название со спецсимволами `<` и `&` → разметка не ломается
16. Одна фраза, отправленная десять раз → **побайтово одинаковый ответ**. Плавающие формулировки означают, что отвечает модель, это ошибка сборки

## 18. Правки после пилота (2026-07-22, по итогам пилота)

Согласованные отклонения от шаблонов §6 — новый эталон для связки:

1. **Вёрстка карточек:** блоки разделяются пустой строкой — [что/когда] · [👥📍📝] · [💻 ссылки] · [👉 календарь].
2. **Правило зон (А):** время без города в ответах на уточнение/перенос трактуется **в зоне календаря**; город всегда побеждает. В сообщениях 6.5/6.6/6.8 — курсивная подсказка «⏱ Время без города — в твоей зоне: …».
3. **Конфликт (6.4):** после пересечений — блок «🟢 Свободно в этот день» с окнами, куда встреча влезает по длительности.
4. **Конференции:** в каждую встречу вшиваются **и Meet, и Zoom** (две строки 💻); при одной ссылке — текст «Подключиться к встрече».
5. **Меню-команды** (код, без модели): /today /free /next /add /tomorrow /week /free_week /free_next /tz /help /new; неизвестные /команды гасятся кодом. Приветствие /start — детерминированная карточка.
6. **Контекст ожидания времени:** пока висит ожидание, любое сообщение с распознаваемым временем — это ответ (даже не reply).
7. Доп. шаблоны: «🔄 Встреча обновлена», «⚠️ Ошибка», «🧹 Сброшено/Чистый лист», «🤷 Не знаю такую команду».
8. Карточка «🔄 перенесена» — полная (👥/📍/📝/💻/👉), как «создана»; «удалена», «не добавлена» и «конфликт» — намеренно короткие.
9. Настройки напоминаний `/reminders` (кнопки, код): дни/время плана дня, время плана недели, ярусы за сутки/час/30/10/5 минут.
10. Ответ на вопрос времени длиннее ~40 символов дополнительно прогоняется через классификатор — детали (длительность/описание/место/участники) вливаются в ожидающую встречу.
